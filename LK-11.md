# Laporan Analisis, Perbaikan Bug, dan Refactoring (LK-11)

## 1. Identifikasi Bug & Masalah (Minor)
Berdasarkan analisis proyek, ditemukan beberapa bug dan isu minor:
1. **Routing Todo List Tidak Terdaftar & Redirect Error**: Controller `TodoController` sudah dibuat, namun route-nya belum didaftarkan di `routes/web.php`. Form dan redirect di dalam aplikasi menggunakan route name seperti `todo.post` dan `todo.delete` yang tidak eksis, sehingga menyebabkan `RouteNotFoundException`.
2. **Fitur Update/Checkbox Selesai Belum Berfungsi (Empty Form)**: Pada view `resources/views/todo/index.blade.php`, terdapat `<form action="#" method="POST">` kosong yang seharusnya berfungsi sebagai checkbox untuk menandai status tugas (is_done).
3. **Penanganan Error Terlalu Luas & Silent Exception**: Pada `TodoController`, beberapa method menggunakan blok `try-catch (\Exception $e)` secara luas. Method `index` bahkan mengembalikan collection kosong (`collect()`) jika terjadi error, yang mana berpotensi menyembunyikan error penting dari database tanpa adanya proses logging.

## 2. Dokumentasi Bug Fix (Sebelum vs Sesudah)

### Bug 1: Routing Todo List Belum Terdaftar & Redirect Error
- **Sebelum**: Rute untuk Todo tidak ada di `web.php`. Di view memanggil `route('todo.post')` dan controller me-redirect ke `route('todo')`.
- **Sesudah**: Rute telah ditambahkan secara benar di `web.php` dan nama rute diperbaiki.
  - Ditambahkan route `todo.store`, `todo.update`, dan `todo.destroy`. Route `/dashboard` dihubungkan langsung ke `TodoController@index`.
  - Di `TodoController`, semua redirect disesuaikan menjadi `redirect()->route('dashboard')`.

### Bug 2: Fitur Update/Checkbox Selesai Belum Berfungsi (Empty Form)
- **Sebelum**: 
  ```html
  <form action="#" method="POST">
      @csrf
  </form>
  ```
- **Sesudah**: Menambahkan form update yang mengarahkan ke `route('todo.update')` menggunakan `@method('PATCH')` dengan checkbox yang langsung disubmit saat diubah statusnya. Selain itu, ditambahkan efek coretan (line-through) saat tugas telah selesai.
  ```html
  <form action="{{ route('todo.update', $todo->id) }}" method="POST">
      @csrf
      @method('PATCH')
      <input type="hidden" name="is_done" value="{{ $todo->is_done ? '0' : '1' }}">
      <input type="checkbox" onchange="this.form.submit()" {{ $todo->is_done ? 'checked' : '' }} class="...">
  </form>
  ```

### Bug 3: Penanganan Error Terlalu Luas (Silent Exception)
- **Sebelum**:
  ```php
  try {
      $todos = Todo::orderBy('created_at', 'desc')->get();
  } catch (\Exception $e) {
      $todos = collect();
  }
  ```
- **Sesudah**: Blok `try-catch` dihapus agar framework Laravel secara natural dapat menangani exception atau error default.
  ```php
  $todos = Todo::orderBy('created_at', 'desc')->get();
  ```

## 3. Refactoring Kode

### Refactoring 1: Implementasi *Implicit Route Model Binding* & Validasi pada Update
- **Deskripsi**: Menyederhanakan pencarian data dengan melakukan binding `Todo $todo` pada parameter method dibandingkan mencari secara manual menggunakan `findOrFail`. Selain itu, ditambahkan aturan validasi yang hilang pada method update.
- **Sebelum**: 
  ```php
  public function update(Request $request, string $id)
  {
      $todo = Todo::findOrFail($id);
      $data = $request->only(['task', 'is_done']);
      // ...
  }
  ```
- **Sesudah**:
  ```php
  public function update(Request $request, Todo $todo)
  {
      $request->validate([
          'task' => 'sometimes|required|min:5|max:25',
          'is_done' => 'sometimes|boolean'
      ]);
      $data = $request->only(['task', 'is_done']);
      // ...
  }
  ```

### Refactoring 2: Standardisasi Penamaan Route (Resource Naming)
- **Deskripsi**: Menyamakan struktur penamaan *route name* agar sesuai dengan standar resource route di ekosistem Laravel.
- **Sebelum**: Menggunakan nama rute yang kurang standar/hardcoded seperti `todo.post` dan `todo.delete`.
- **Sesudah**: Nama route di-update di Controller dan View menjadi format standar: `todo.store`, `todo.update`, dan `todo.destroy`.
